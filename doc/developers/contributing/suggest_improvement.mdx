# Suggest Improvement

Use this path for improvements to existing behavior (usability, performance, docs, reliability, developer experience).

Submit your proposal on:

- [LiRAYS-SCADA Issues](https://github.com/LiRAYSgj/LiRAYS-Scada/issues)

## Before You Open An Improvement Proposal

- Check whether the topic already exists in open or closed issues.
- Clearly describe the current limitation.
- Keep scope focused to one improvement per issue.

## What To Include

1. **Title**: concise and outcome-oriented.
2. **Current behavior**: what happens today and why it is limiting.
3. **Proposed improvement**: what should change.
4. **Why it helps**: operator/integrator/developer value.
5. **Alternatives considered**: other options and trade-offs.
6. **Compatibility impact**:
   - breaking/non-breaking
   - migration needed or not
7. **Acceptance criteria**: concrete conditions to consider it complete.

## Tips

- Prefer measurable outcomes over implementation details.
- If possible, include a small mockup or example output.
