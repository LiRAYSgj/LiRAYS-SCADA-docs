# Suggest New Feature

Use this path for net-new capabilities not currently provided by LiRAYS-SCADA.

Open your feature request on:

- [LiRAYS-SCADA Issues](https://github.com/LiRAYSgj/LiRAYS-Scada/issues)

## Before You Open A Feature Request

- Search existing feature requests first.
- Validate the request as a new capability (not an improvement to an existing one).
- Focus each issue on one feature idea.

## What To Include

1. **Feature summary**: one paragraph describing the capability.
2. **Problem/use case**: who needs it and what problem it solves.
3. **Proposed behavior**: expected UX/API/protocol behavior.
4. **Scope and non-goals**:
   - in scope
   - explicitly out of scope
5. **Impact analysis**:
   - performance/resource impact
   - security implications
   - operational complexity
6. **Compatibility and migration**:
   - breaking changes?
   - fallback/migration path?
7. **Acceptance criteria**: testable completion definition.

## Tips

- Prefer clear problem statements and success criteria over broad ideas.
- Attach example workflows or diagrams when useful.
