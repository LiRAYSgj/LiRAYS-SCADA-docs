Clients and Operational CLI

lirays Operational CLI (src/liraysctl/*)

The CLI is an operator/admin control plane for local host maintenance.

Command families:

  • user lifecycle: create/remove/update admin/operator
  • PAT lifecycle: generate/list/revoke
  • settings inspection/update
  • service restart helpers (systemd/launchd)
  • metrics watch helper

Key architecture traits:

  • Reads runtime settings using same settings resolver model.
  • Opens static.db directly for user/PAT operations.
  • Uses interactive prompts for secrets and token metadata.
  • Encourages explicit operational workflows (for example, bootstrap URL after admin removal).

Rust SDK (clients/rust-client)

Design:

  • Async command client over websocket + protobuf.
  • Correlates command responses using per-command ids and pending oneshot map.
  • Applies command timeout per request.
  • Exposes typed helper APIs for create/list/set/get/edit/bulk operations.

Subscription model:

  • Opens a separate websocket for SUB streams.
  • Returns async stream of (var_id, typed_value) tuples.

Connection modes:

  • anonymous (connect)
  • PAT-authenticated (connect_with_pat)
  • explicit options (connect_with_options)

Python SDK (clients/python-client)

Design:

  • PyO3 extension module wrapping local Rust SDK crate.
  • Blocking Python API, async work executed on internal Tokio runtime.
  • Mirrors Rust SDK surface with Pythonic classes and typed stubs.

This avoids duplicating protocol logic and keeps Rust SDK as protocol implementation source.

When You Add A New Protocol Capability

Update sequence should be treated as mandatory checklist:

  1. proto/namespace/*.proto
  2. server implementation (src/rtdata/variable.rs)
  3. frontend command builder and WS client
  4. Rust SDK command/protocol wrappers
  5. Python bindings surface if externally needed
  6. docs and demos

Skipping any of these usually creates integration drift.

Integrator Guidance

  • Prefer PAT auth for service-to-service access.
  • Treat timeout handling as part of normal operation (network/restart aware).
  • Use list/get before set in automation flows when type/constraint certainty matters.
  • Keep SDK and server versions aligned with shared proto schema expectations.
