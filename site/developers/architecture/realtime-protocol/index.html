Realtime Protocol and Engine

Contract Source Of Truth

The canonical wire contract is in proto/namespace/*.proto:

  • commands.proto: command and response envelopes
  • types.proto: namespace/value/domain payloads
  • events.proto: pushed events
  • enums.proto: enum values for status/types/events

Rust backend and Rust SDK compile these protos via prost-build; frontend compiles TypeScript bindings via ts-proto.

WebSocket Semantics

Endpoint: /ws

  • Command input formats:
  • protobuf binary frames
  • JSON text frames
  • Response output mirrors input format style (binary for protobuf command, JSON text for JSON command).
  • Event pushes are emitted continuously for subscribed sessions.

Command Surface

Supported commands in VariableManager::exec_cmd:

  • Namespace mutation: Add, Del, AddBulk, EditMeta
  • Namespace/value query: List, Get
  • Value write: Set
  • Streaming control: Sub, Unsub

Response envelope always includes:

  • typed response payload (matching command)
  • OperationStatus
  • optional error_msg

Authorization Matrix

  • Admin-only commands: Add, Del, AddBulk, EditMeta
  • Admin and Operator: List, Get, Set, Sub, Unsub

Unauthorized calls return error responses; frontend and SDKs surface these as operation failures.

Internal Storage Model

VariableManager keeps three core structures:

  • items_tree (sled): namespace metadata keys parent/\0name
  • values_tree (sled): persisted variable values keyed by full variable path
  • values_cache (in-memory): hot working set with dtype and constraints

Write path is write-back:

  1. Set mutates values_cache
  2. value is marked dirty in dirty_values
  3. periodic loop batches dirty entries to values_tree

Event Fan-Out

Subscribers register per-session bounded channels.

  • Broadcast is non-blocking.
  • Slow/full listeners drop batches instead of backpressuring producers.
  • Closed listeners are evicted.

Event filtering is per connection:

  • VarValueEv sent only for subscribed ids
  • TreeChangedEv sent only when tree-change subscription is active

Tree Change Model

Namespace mutations emit TreeChanged events with FolderChanged entries.

  • Incremental mode: removed ids + new folders + new variables
  • Reload mode: reload=true tells clients to refresh parent folder content

Frontend tree reconciliation logic is built around this dual mode.

Constraint Validation

Before accepting new values:

  • Numeric: dtype checks + min/max bounds
  • Text: options whitelist + max length
  • Boolean: bool/int/float coercion rules

Validation is centralized in validate_constraints.

Cross-Layer Protocol Gotcha

Response and Event protobuf envelopes share overlapping wire tags. Frontend TagStreamClient handles this by decoding as Response first and falling back to Event when no pending command matches cmd_id.

If you evolve proto envelope fields, verify this logic still holds.

Adding A New Realtime Command

  1. Add command/response in proto.
  2. Rebuild generated code in backend/frontend/SDKs.
  3. Implement server command branch in VariableManager::exec_cmd.
  4. Add frontend command builder and client handling.
  5. Add SDK support in Rust client and Python binding if needed.
  6. Add tests for auth, status mapping, and timeout/error paths.
