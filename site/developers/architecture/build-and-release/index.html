Build, Packaging, and Release Architecture

Build Dependency Chain

Production backend binary embeds frontend static assets at compile time.

Required order:

  1. Generate frontend TS protos (frontend npm run generate:proto).
  2. Build frontend (frontend/build).
  3. Build Rust backend (cargo build --release).

src/http/resources/static_files/service.rs uses include_dir! on frontend/build, so missing frontend build artifacts will break or stale the embedded UI.

Tooling Surface

Primary orchestrator: Makefile

Notable targets:

  • app build/test: frontend, test
  • packaging: deb, rpm, mac
  • SDK docs/publish: rust-client-*, python-client-*

Docker Build

Dockerfile uses 3 stages:

  1. frontend builder (Node 24 + protobuf)
  2. backend builder (Rust + embedded frontend build output)
  3. alpine runtime image (lirays-scada + lirays)

This produces a single-container runtime with embedded web UI.

OS Packaging Architecture

Debian (packaging/debian)

  • Installs binaries + systemd service + default settings.
  • Post-install creates service user/group, data dir, and seeds settings.yaml from default if missing.
  • Service runs with dedicated user and strict filesystem write path.

RPM (packaging/rpm)

  • Similar service model to Debian.
  • %post handles service enable/start and first-time settings seeding.
  • Includes dedicated lirays service account setup in %pre.

macOS (packaging/macos)

  • Builds universal binary package (pkgbuild/productbuild).
  • Installs LaunchDaemon plist + binaries + settings defaults.
  • Postinstall script bootstraps and kickstarts daemon.

Runtime Service Model

Packaged service managers call:

  • server binary with explicit config path
  • persistent data directory outside binary location
  • restart-on-failure behavior

This keeps upgrades mostly config/data preserving.

Release Artifacts

Output directory: distributions/

Contains platform artifacts (.deb, .rpm, .pkg) produced by make targets.

Safe Release Checklist

  1. Rebuild frontend and backend from clean state.
  2. Run frontend unit tests and Rust tests.
  3. Verify protocol compatibility for frontend + SDKs.
  4. Validate packaging scripts against current config defaults.
  5. Smoke-test auth-enabled and auth-disabled startup modes.
  6. Verify service restart behavior on each target platform.
