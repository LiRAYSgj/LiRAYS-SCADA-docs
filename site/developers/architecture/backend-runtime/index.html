Backend Runtime Architecture

Entrypoints

  • Runtime binary: src/main.rs (lirays-scada)
  • CLI binary: src/bin/liraysctl.rs (lirays)

The server path is:

main -> app::run -> http::run_http_server

Runtime Composition (AppState)

AppState is the shared dependency container injected into Axum handlers and middleware:

  • views: ViewManager (SQLite)
  • users: UserManager (SQLite + Argon2)
  • patoken_manager: PATokenManager (SQLite + Argon2)
  • auth: AuthConfig (enabled flag, secret, TTLs, secure cookie toggle)
  • var_manager: VariableManager (sled realtime namespace/value engine)
  • session_manager: SessionManager (sled + JWT-based session lifecycle)

Route Mounting Strategy

Server routes are mounted in order:

  1. /ws (auth-gated when enabled)
  2. /api/views* (mix of authenticated/admin-only)
  3. /api/auth/*
  4. /api-docs/openapi.json and /swagger
  5. static fallback for all unmatched routes

The final fallback serves embedded frontend assets; this makes backend the single public entrypoint.

Resource Modules

views

  • Owns view CRUD, pagination, sorting, search, and entry-point selection.
  • Maintains invariant that exactly one entry-point view exists.
  • Initializes DB with a default Root View if table is empty.
  • Returns envelope responses and OpenAPI annotations.

auth

  • /api/auth/setup: bootstrap first admin when auth is enabled and admin does not exist.
  • /api/auth/login: username/password login -> session cookie.
  • /api/auth/status: route guards use this to drive redirects.
  • /api/auth/logout: revokes session.

docs

  • Serves OpenAPI JSON generated via utoipa.
  • Serves Swagger UI HTML (CDN-hosted swagger assets).

static_files

  • Serves compile-time embedded files from frontend/build (include_dir).
  • Includes SPA fallback behavior (index.html) for deep links.

user and patoken

  • Provide schema and manager logic.
  • No public PAT CRUD HTTP endpoints yet; PAT lifecycle is handled via lirays CLI.

Middleware Design

Two middleware layers enforce auth at route boundaries:

  • authenticated_only: any authenticated role (admin/operator)
  • admin_only: admin role only

Role extraction order:

  1. Authorization: Bearer <pat> (PAT verification)
  2. lirays_session cookie (access token)
  3. Session refresh flow using refresh token if access token expired

If auth is disabled, middleware passes through.

Background Processes

Launched during startup:

  • Variable dirty flush loop (flush_ms, min 5s)
  • Session purge loop (every 12 hours)
  • Metrics writer loop (every 5 seconds, when enabled)

These loops are intentionally detached from request handlers.

Shutdown Behavior

Graceful shutdown listens for SIGTERM/SIGINT (or Ctrl+C fallback), flushes dirty realtime values, and then exits. This avoids recent value loss in the write-back cache model.

Adding A New HTTP Domain

  1. Create src/http/resources/<domain>/ with mod.rs, service, model, schema init.
  2. Add ensure_schema call in http::ensure_schema if persistent.
  3. Mount routes in run_http_server with proper middleware.
  4. Extend OpenAPI docs if externally visible.
  5. Update frontend/API client layers if user-facing.
