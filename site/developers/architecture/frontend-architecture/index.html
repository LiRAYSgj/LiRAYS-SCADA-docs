Frontend Architecture

Runtime Model

Frontend is a SvelteKit SPA (ssr = false, prerender = true) built to frontend/build and embedded by backend.

In local dev, Vite proxies:

  • /api -> backend HTTP
  • /ws -> backend websocket

Route Groups

  • Root +layout.svelte: global theme/snackbar/navigation-progress shell.
  • (guest) routes: /auth/login, /auth/setup.
  • (app) routes: designer workspace (views list + editor).
  • runtime/* routes: runtime-only display mode for operators and production playback.

Guards call /api/auth/status and redirect according to:

  • auth enabled/disabled
  • admin bootstrap state
  • authenticated vs anonymous
  • operator vs admin role

Workspace Shell ((app)/+layout.svelte)

This is the orchestration hub for designer mode:

  • Left pane: VariableTree
  • Right pane: views list or view editor
  • Graph canvas: @xyflow/svelte
  • Inspector pane: binding and node metadata editing
  • Dialogs: view removal, tree deletion, namespace builder, leave-view guard

Primary UI states:

  • rightPaneMode: views-list or view-editor (route-derived)
  • canvasMode: edit or play
  • multi-selection mode for bulk tree deletion

Data Channels

REST

Used mainly for view management and auth status/logout:

  • /api/views*
  • /api/auth/status
  • /api/auth/logout

All API calls go through apiFetch, which centralizes API-path 401 handling.

WebSocket

TagStreamClient is the single transport for:

  • Tree namespace operations (LIST, ADD, DEL, EDIT_META, ADD_BULK)
  • Realtime value subscription (SUB/UNSUB + push events)
  • Value writes (SET)

createPageTagRealtimeProvider wraps this client with page-level active/id-tracking semantics.

Tree Architecture

Core tree modules:

  • tree-store.ts: normalized state (nodes map + root ids + expanded/loading/errored sets)
  • server-adapter.ts: maps backend ListResponse into frontend TreeNode
  • tree-remote-reconcile.ts: applies pushed TreeChanged events incrementally or by reload request
  • VariableTree.svelte: virtualization, keyboard navigation, dialogs, selection UX

Tree updates are eventually consistent with realtime events, without full refresh unless requested.

Graph + Widget Architecture

Graph nodes represent widget assets with tag bindings.

  • assets/registry.ts: runtime asset registry
  • PlantAssetNode.svelte: dispatches to Svelte or custom-element runtime
  • CustomElementAssetHost.svelte: bridges Svelte node data to web components
  • scada/plugins/internal-widgets/*: built-in widget plugin declarations and custom-element runtime shell

Live values flow through live-utils.ts, which computes tracked tag ids and patches only impacted nodes when possible.

Designer vs Runtime Separation

  • Designer (/views/:id): edit graph, assign bindings, manage namespace.
  • Runtime (/runtime/:id): read-only graph layout + realtime values + allowed writes per widget binding permissions.

Both share rendering primitives; runtime strips editing affordances and always enables realtime provider.

Frontend Extension Points

  • New widget type: add declaration under internal-widgets/widgets, ensure manifest bindings and runtime registration.
  • New backend command usage: add builder in core/ws/command-ws-client.ts and handling in TagStreamClient.
  • New page-level mode: add route group and guard updates in layout loads.
