Architecture Guide

This section is the main technical map of LiRAYS-SCADA for future maintainers. It explains how the backend, frontend, protocol, persistence, authentication, SDKs, and packaging fit together so you can make safe changes without reverse-engineering the repo every time.

How To Use This Guide

  1. Start with System Overview to understand boundaries and runtime topology.
  2. Read Backend Runtime Architecture and Realtime Protocol and Engine before changing server behavior.
  3. Read Frontend Architecture before touching Svelte route/layout logic, tree behavior, or graph runtime.
  4. Read Data and Persistence Architecture and Authentication and Security Architecture before any schema, auth, or storage changes.
  5. Use Clients and Operational CLI and Build, Packaging, and Release Architecture when changes affect SDKs, tooling, or distribution.

Change Impact Quick Matrix

  • Realtime command behavior changed: backend src/rtdata/*, frontend frontend/src/lib/core/ws/*, SDKs in clients/*.
  • View CRUD/API changed: backend src/http/resources/views/*, frontend frontend/src/lib/features/views/*.
  • Auth/session behavior changed: backend src/http/resources/auth/*, frontend route guards and auth pages.
  • Data model changed: src/http/resources/*/model.rs, migration strategy in managers, docs in this folder.
  • Deployment defaults changed: settings-dev.yaml, packaging/*, Dockerfile, Makefile.
